# Сущности

Опишем сущности нашей программы

## Окна и контроллеры

1 основное окно, переходов между окнами нет, есть высплывающие системные окна (для выбора директории).

Для основного окна 1 контроллер.

## Сервисы

### Сервис для поиска файлов

Проверяет исходную и результирующие директории и составляет план для обработки файлов.

**Аргументы**:
- Исходная директория
- Результирующая директория
- Правило разрешения конфликтов

**Результат работы**: набор заданий, который состоит из исходного файла и резльтирующего файла.

### Сервис для обработки файлов

Принимает исходный файл, создает буффер и временный файл, производит работу над исходным файлом и записывает результат в временный файл.

**Аргументы**:
- Исходный файл
- Временный файл
- Xor-маска 

**Резльтат работы**: записанный результат во временный файл.

### Оркестратор

Проверяет аргументы для работы программы.
После кнопки `Старт` -- запускает сервис для поиска файлов, принимает структуру. Последовательно запускает обработку файлов.
После обработки файла записывает результат из временного файла в результирующий с соблюдением состояния программы.
При кнопках остановки и отмены -- изменяет состояние работы.

**Аргументы**: нет.

**Результат работы**: готовые файлы.

## Утилиты

### FileProcessor

Утилита для создания временных файлов, записи, удаления, изменения файлов.

## Структуры

Вспомогательные структуры -- `TimerArgs`, `FileDuplicationRule`.
Эти структуры -- это удобное представление входных данных.

## Состояние

Состояние приложения хранится в оркестраторе, им является `WorkingState`.
Каждое изменение этого состояния вызывает сигнал для обновления состояния UI.

Состояние сервисов -- `ServiceState` -- внутреннее состосяние работы сервиса.
Эти состояния не влияют на UI, нужны в основном для формальной верификации переходов.
