# Сущности

Опишем сущности нашей программы

## Окна и контроллеры

1 основное окно, переходов между окнами нет, есть высплывающие системные окна (для выбора директории).

Для основного окна 1 контроллер.


## Состояние

Состояние приложения будет в синглтоне AppState.  
Состояние содержит в себе:
- Маска файлов
- Исходная директория
- Результирующая директория
- Флаг удаления исходных файлов
- Аргумент таймера -- структура из флага и периода таймера.
- Правило для повторяющихся имен файлов
- Количество найденных файлов
- Номер текущего файла / количество обработанных файлов
- Оффсет текущего файла (при остановке процесса)
- Xor-маска (8 байт)
- Состояние работы -- Idle, Running...

## Сервисы

### Сервис для поиска файлов

Проверяет исходную и результирующие директории и составляет мапу для обработки файлов.

**Аргументы**:
- Исходная директория
- Результирующая директория

**Результат работы**: структура (unordered map), которая хранит key исходный файл, value итоговый файл.

### Сервис для обработки файлов

Принимает исходный файл, создает буффер и временный файл, производит работу над исходным файлом и записывает результат в временный файл.

**Аргументы**:
- Исходный файл
- Xor-маска 

**Резльтат работы**: временный файл.

### Оркестратор

Проверяет аргументы для работы программы.
После кнопки `Старт` -- запускает сервис для поиска файлов, принимает структуру. Последовательно запускает обработку файлов.
После обработки файла записывает результат из временного файла в результирующий с соблюдением состояния программы.
При кнопках остановки и отмены -- изменяет состояние работы.

**Аргументы**: нет.

**Результат работы**: готовые файлы.
